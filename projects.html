Project 1

   Branch Predictor using VHDL and ModelSim

Description

My contribution in this project is underling the results and main differences among the three branch predictors. The outlined differences can make an impact on the latency of the CPU by having more accurate predictions. I have explored and understand how data stalls can be better avoid by having simple mechanism to take advantage of running time outcomes of branches. I want to present the results of these differences and make a clear comparison to know which predictor will be a better implementation. This project entails the hardware architecture of predictors and how this could help manage the flow of data. Unlike a static predictor on which we use a fixed prediction each time, a dynamic predictor does the prediction at running time. This is beneficial in a sense that the predictor is adapting to the behavior of the program. Accuracy and latency are important aspects that predictors can enhance since if a prediction is made correct it will impact the performance of the CPU.

Link to code: https://github.com/pcuji/branch_predictor.

Project 2

   Decentralized Peer to Peer File Sharing using C++

Description

This project is underling the results and performance of the Decentralized file sharing and the step by step procedure and implementation of a Decentralized peer to peer architecture. The architecture is popular since it allows downloads files of any other peer and also search multiple index servers with lower latency for file sharing. This description of the decentralized file sharing relies on a model that consist of nodes and an index server. The nodes will register with a random server and it will provide the information as the ID and the files that they possess to share with others nodes/peers and server indexes. The index server will collect the information of all this peers and their files. It will also possess a function that will allow peers to request information about the files. When the peer that request it will loop over all index servers and index tables table until it finds the file. The server will return the peers-nodes information to the requester. At this point the peer/node can choose to download form the peer that contains the file.

Introduction

This File Sharing system provided file sharing among nodes or peers. The mainframe architecture of the decentralized file sharing relies on an architecture which allows the peer to peer (P2P) download and search into the servers to be independent of each other. The architecture of this system is a basic architecture of networks and technology in today infrastructure.

Link to code: https://github.com/pcuji/peer_to_peer_file_sharing.
